🌟 Résumé du calcul effectué dans CombiCheckerComponent

Le composant vérifie toutes les combinaisons possibles de 2 ou 3 nombres dans le tableau A et compte celles dont la somme est présente dans le tableau B.

🔍 Détails du calcul

  1. Données :
    A : tableau de nombres saisis par l’utilisateur.
    B : tableau de nombres saisis par l’utilisateur.
  2. Objectif :
    Trouver toutes les paires et triplets de nombres de A dont la somme est exactement égale à un élément de B.

🧮 Étapes mathématiques

📌 Étape 1 : Combinaisons de 2 éléments

Pour chaque paire (A[i], A[j]) avec i < j :

Calculer leur somme : sum2 = A[i] + A[j]
Vérifier si sum2 ∈ B.

Exemple : A = [1,2,3], B = [4,5]

  • 1+2=3 (non dans B)
  • 1+3=4 (dans B ✅)
  • 2+3=5 (dans B ✅)

📌 Étape 2 : Combinaisons de 3 éléments

Pour chaque triplet (A[i], A[j], A[k]) avec i < j < k :

Calculer leur somme : sum3 = A[i] + A[j] + A[k]
Vérifier si sum3 ∈ B.

Exemple : A = [1,2,3,4], B = [6,7]

  • 1+2+3=6 (dans B ✅)
  • 1+2+4=7 (dans B ✅)
  • 1+3+4=8 (non dans B)
  • 2+3+4=9 (non dans B)

🔢 Autres informations importantes :

  • totalTested : nombre total de combinaisons testées.
  • validCombinations : liste des combinaisons valides.
  • result : nombre total de combinaisons valides.

📈 Complexité mathématique :

  • Combinaisons de 2 éléments : n(n-1)/2
  • Combinaisons de 3 éléments : n(n-1)(n-2)/6

📝 Résumé simple :

Tu saisis des nombres dans A et B.
Le programme vérifie toutes les sous-listes de 2 ou 3 nombres de A.
Pour chaque sous-liste, il calcule la somme.
Si la somme est dans B, c’est une combinaison gagnante.
Il affiche le nombre total de combinaisons valides et les détaille.
